Bases: object

Handles the S3 export process, enabling the downloading, parsing, and management of data from an Amazon Bedrock Knowledge Base. The class provides functionalities to process and save data chunks and source documents in a structured local directory.

It is equipped with configurations to support selective operations, such as filtering by S3 key prefix, including additional metadata, and optional embedding integration.

Variables:

  • bucket_name – Name of the S3 bucket containing the Knowledge Base data.

  • key_prefix – Prefix to filter the S3 objects in the specified bucket.

  • region – AWS region containing the S3 bucket.

  • limit – Maximum number of S3 objects to process (-1 for no limit).

  • output_dir – Directory path where output files will be written.

  • s3_client – AWS S3 client used for interacting with the S3 service.

  • id_rewriter – Facilitates rewriting unique IDs with a custom tenant identifier.

  • metadata_fn – Optional function to produce or modify object metadata.

  • include_embeddings – Specifies whether embeddings should be included.

  • include_source_doc – Specifies whether to include source documents in processing.

Initializes an instance of the class, setting up configuration parameters such as the AWS S3 bucket details, file processing configurations, and optional metadata handling.

Parameters:

  • region – The AWS region for the S3 bucket.

  • bucket_name – The name of the AWS S3 bucket.

  • key_prefix – The prefix path in the S3 bucket from where data will be accessed.

  • limit – The maximum number of objects to process from the S3 bucket. Defaults to -1, which implies no limit.

  • output_dir – The directory to which the output files will be saved locally.

  • metadata_fn – A callable function that accepts a string and returns a dictionary containing metadata for processing files.

  • include_embeddings – Determines whether to include embeddings in the processing. Defaults to True.

  • include_source_doc – Specifies whether to include source documents during processing. Defaults to False.

  • tenant_id – Identifier for the tenant for managing tenant-specific configurations. Can be None.

  • kwargs – Additional keyword arguments for custom processing configurations.

Initializes an instance of the class responsible for handling a directory of chunks, a specific collection identifier, and optional metadata keys. The constructor sets up the necessary parameters and ensures the storage directory is prepared for further operations. A collection identifier can optionally be provided, but if not, it defaults to a timestamp.

Parameters:

  • metadata_keys (Optional[List[str]]) – A list of keys for associating metadata with collection items. These keys allow additional information to be attached to the handled objects.

class graphrag_toolkit.lexical_graph.indexing.load.s3_based_chunks.S3BasedChunks(region: str, bucket_name: str, key_prefix: str, collection_id: str | None = None, s3_encryption_key_id: str | None = None, metadata_keys: List[str] | None = None)[source]¶

Bases: NodeHandler

Manages the interaction with an S3-based storage system to handle chunks of data, providing functionality for serialization, metadata filtering, iteration, and upload operations.

This class is intended for managing data chunks in an S3 storage environment through a defined bucket, key prefix, and optional collection ID. It allows for filtering metadata, processing nodes, server-side encryption during data upload, and custom iteration functionality.

Variables:

  • region – Represents the AWS region associated with the S3 service.

  • bucket_name – The name of the S3 bucket being used for data storage.

  • key_prefix – The prefix or folder structure for grouping items in S3.

  • collection_id – A unique identifier for referring to a specific data collection.

  • s3_encryption_key_id – Optional KMS encryption key ID for securing S3 objects.

  • metadata_keys – Optional list defining metadata keys to retain for filtering.

Initializes an instance with the given parameters for file storage or data collection configuration. This initialization sets up region-specific S3 bucket information, encryption, data organization prefixes, and metadata keys for efficient management.

Parameters:

  • region (str) – The AWS region where the S3 bucket is located.

  • bucket_name (str) – The name of the S3 bucket being accessed.

  • key_prefix (str) – The prefix to be used for objects stored in the S3 bucket.

  • collection_id (Optional[str]) – The identifier for a specific data collection. Defaults to a timestamp (‘%Y%m%d-%H%M%S’) if not provided.

  • s3_encryption_key_id (Optional[str]) – The ID of the key used for encryption in S3. This is optional and allows for securing stored objects.

  • metadata_keys (Optional[List[str]]) – A list of metadata keys that are relevant for the storage process or data organization.

accept(nodes: List[BaseNode], **kwargs: Any) → Generator[BaseNode, None, None][source]¶

Generates nodes from the input list, processes them, and writes chunks containing their metadata to Amazon S3 based on the provided configuration.

This function iterates over the given nodes, examining their metadata to check for a required key. If the key does not exist in the node metadata, the node is serialized in JSON format and uploaded to the configured S3 bucket. The node’s corresponding metadata chunk is stored using either AWS KMS encryption or AES256 depending on the given configuration. Finally, it yields each node after processing.

Parameters:

  • nodes – A list of nodes to process and potentially upload to S3. Each node in the list should conform to the expected structure and type defined by BaseNode.

  • kwargs – Additional keyword arguments that may be used for future extension or additional processing in this function.

Returns:

A generator that yields each processed node from the input list sequentially, after its storage is handled (if applicable).
